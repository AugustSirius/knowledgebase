# 学术演讲完美交付实战剧本

## 1. 场景分析

### 为什么这个场景至关重要？

**潜在影响的三个维度：**

1. **科研声誉资本积累**
   - 一次优秀的talk可以让你在该领域被记住18-24个月
   - 会后私下交流往往产生50%以上的合作机会
   - 被邀请做talk本身就是信号，糟糕的交付会损害邀请者的声誉（他们不会再推荐你）

2. **创业资源获取**
   - 学术圈是高质量人才的蓄水池
   - 技术顾问、早期用户、潜在投资人线索往往来自学术网络
   - 一个清晰的技术叙事可以转化为融资pitch的核心

3. **跨圈层影响力放大**
   - 学术talk的录像/slides会被分享，形成长尾传播
   - 建立"既懂科研又懂商业"的稀缺人设
   - 为未来的invited talk、panel、媒体采访建立素材库

### 常见的错误假设和期望偏差

**错误假设1：** "听众都是专家，我要展示技术深度"
- **现实：** 即使是专业会议，80%的听众对你的具体方法不熟悉，他们想知道"这为什么重要"和"我能用吗"

**错误假设2：** "20分钟很短，我要塞满信息"
- **现实：** 人只能记住3个要点。信息过载导致什么都记不住。

**错误假设3：** "Q&A是测试环节，我要证明我什么都知道"
- **现实：** Q&A是建立连接的黄金时刻，"好问题，我们也在探索"比强行回答更有效。

**错误假设4：** "只要科学内容扎实就够了"
- **现实：** 交付质量=内容×表达×能量。内容只占1/3。

**期望偏差：**
- 你期望：传递完整的技术细节
- 听众期望：获得启发、找到连接点、评估你是否值得深入交流

### 成功的关键要素

**要素1：钩子的即时性（前30秒）**
- 听众在30秒内决定是否投入注意力
- 需要一个具体、可视化、反直觉或情感化的开场

**要素2：叙事的单线程（结构）**
- 一个核心故事线，所有内容都服务于这条线
- 避免分支和"顺便说一下"

**要素3：能量的可持续性（交付）**
- 20分钟保持一致的能量水平
- 声音、肢体、节奏的变化

**要素4：互动的预设性（Q&A）**
- 提前设计你希望被问到的问题
- 在talk中埋下"钩子"引导提问方向

**要素5：后续的可操作性（转化）**
- 每个听众都应该清楚"如果我想了解更多，下一步是什么"

## 2. 准备清单

### 事前必须准备的事项（倒推时间线）

**T-14天：情报收集**
- [ ] 确认听众构成（学生/教授/工业界比例）
- [ ] 了解会议主题和其他speaker话题（避免重复，找到定位）
- [ ] 询问组织者：最希望我讲什么？过往哪个talk最受欢迎？
- [ ] 查看会议往年视频，了解氛围和提问风格

**T-10天：内容框架**
- [ ] 写出一句话核心信息（"如果只能记住一件事"）
- [ ] 设计钩子的3个备选方案
- [ ] 绘制故事线流程图（6-8个关键节点）
- [ ] 确定"删除候选"（准备30%可以根据时间删减的内容）

**T-7天：Slides初稿**
- [ ] 制作slides（15-20张，含标题页和致谢页）
- [ ] 每张slide的停留时间标注（目标：1-1.5分钟/张）
- [ ] 准备"备用slides"（放在附录，应对可能的深入提问）
- [ ] 视觉检查：6米外能否看清所有文字和图表

**T-5天：预演1.0**
- [ ] 对着镜子/录像完整讲一遍
- [ ] 计时（目标：17-18分钟，留2-3分钟buffer）
- [ ] 标记卡壳的地方
- [ ] 找1-2个同事做试讲（不同背景最佳）

**T-3天：迭代优化**
- [ ] 根据反馈修改slides和话术
- [ ] 准备Q&A的10个预设问题和答案（写下来）
- [ ] 设计"如果超时"的快速收尾方案
- [ ] 准备一句话自我介绍（3个版本：10秒/30秒/1分钟）

**T-1天：最终准备**
- [ ] 完整预演一次（计时，不中断）
- [ ] 准备物料：名片、补充材料的QR码、demo视频备份
- [ ] 检查技术：翻页笔电池、适配器、slides在U盘和云端双备份
- [ ] 设定手机提醒：提前15分钟到场

**T-2小时：现场准备**
- [ ] 测试投影和翻页笔
- [ ] 走到会场最后一排，检查可见性
- [ ] 测试麦克风音量和移动范围
- [ ] 喝水，上洗手间，做3次深呼吸

### 需要收集的信息

**关于听众：**
```
1. 主要背景领域：________
2. 职业阶段分布：博士生__% 博后__% PI__% 工业界__%
3. 对我的话题的熟悉度：1(完全陌生)-5(非常熟悉) = __
4. 他们最关心的问题：________
5. 潜在的合作者/招聘对象/客户：________
```

**关于会议：**
```
1. 会议规模：__人
2. 我的时间段：第__个speaker，上一个讲什么：________
3. 房间布局：剧院式/教室式/圆桌，容量__人
4. 技术设置：有无激光笔、倒计时提醒、录像
5. 社交安排：会前coffee break/会后dinner
```

**关于组织者：**
```
1. 邀请我的原因：________
2. 他们的期望：________
3. 我可以如何帮助他们（让这次邀请物超所值）：________
```

### 心理准备和心态调整

**心态框架：**
> "我不是来证明自己的，我是来分享一个有趣故事并建立连接的。"

**能量管理：**
- **前一晚：** 早睡，避免酒精，准备好衣服
- **当天早上：** 轻运动（10分钟），高蛋白早餐，避免过多咖啡因
- **演讲前1小时：** 独处时间，听激励音乐，回顾核心信息（不是背稿）
- **演讲前10分钟：** 肢体扩展动作（power pose 2分钟），微笑练习

**压力重构：**
- 紧张=兴奋。告诉自己"我很兴奋"而不是"我很紧张"
- 可视化：想象一个听众在会后说"这太有启发了"
- 最坏情况预演：即使完全搞砸，明天太阳还会升起

**身份定位：**
```
我是：
- 一个有独特视角的研究者（科研身份）
- 一个正在解决真实问题的创业者（商业身份）
- 一个连接学术和应用的桥梁（跨界身份）

我不是：
- 无所不知的专家
- 需要取悦所有人的表演者
```

技术物料：

- **笔记：** 会后立即记录的模板（见后续跟进部分）

## 3. 分步执行指南

### 第一步：到场与环境掌控（T-15分钟）

**时间节点：** 演讲开始前15分钟到达会场

**具体做法：**
1. 找到组织者，确认流程和时间
2. 测试技术设备（投影、翻页笔、麦克风）
3. 走到会场最后一排，检查slides可见性
4. 观察前一个speaker的风格和听众反应
5. 找到"安全岛"（会后可以快速撤退或深入交流的位置）

**注意事项：**
- 不要在演讲前5分钟还在改slides
- 如果技术有问题，立即找组织者，不要自己折腾
- 观察听众的能量水平（如果是下午最后一场，需要更多能量）

---

### 第二步：开场钩子（前30秒）

**时间节点：** 被介绍后，走上台到第一张内容slide

**具体做法：**

**动作序列：**
1. 走上台（步伐稳定，不要跑）
2. 站定，深呼吸，微笑扫视全场2秒
3. 说第一句话（声音比平时大20%）

**钩子模板（选择一个）：**

**版本A：问题式（适合方法创新）**
```
"在座有多少人曾经尝试用AI预测蛋白质结构？
[停顿，观察举手]
那么有多少人在真实实验中成功应用了这些预测？
[停顿]
这个巨大的落差，就是我们今天要解决的问题。"
```

**版本B：故事式（适合应用导向）**
```
"三个月前，一位癌症研究者给我发邮件说：
'你的工具在20分钟内找到了我们团队花3年没找到的靶点。'
今天我想分享，我们是如何做到的。"
```

**版本C：数据冲击式（适合结果导向）**
```
"[显示一张对比图]
左边：传统方法，6个月，成功率15%
右边：我们的方法，2天，成功率78%
这不是渐进式改进，这是范式转变。"
```

**版本D：反直觉式（适合挑战常识）**
```
"我们都被教导：更多数据=更好的模型
但在生物AI中，我们发现：
有时候，更少的数据反而能得到更好的结果。
这听起来很疯狂，让我解释为什么。"
```

**话术要点：**
- 第一句话必须完整、清晰、无需背景知识
- 使用"你/我们"而不是"研究表明"
- 包含一个视觉元素（图/数字/对比）
- 设置一个"认知缺口"（引发好奇）

**注意事项：**
- 不要从"感谢组织者"开始（可以在第2张slide快速致谢）
- 不要道歉（"我准备得不够充分"）
- 不要从agenda开始（无聊）
- 如果有笑话，必须是自然的，不要强求

---

### 第三步：背景设定（2-3分钟）

**时间节点：** 钩子之后，进入第2-3张slides

**具体做法：**

**结构：问题→重要性→现有方案的局限**

**Slide 2：问题定义**
```
标题："The Challenge"或"Why This Matters"

内容框架：
- 一句话问题陈述
- 一个具体场景/例子（让非专家也能理解）
- 一个数字（市场规模/影响人数/时间成本）

示例：
"药物发现平均需要10年，成功率不到10%
主要瓶颈：我们无法准确预测分子如何与靶点结合
[图示：传统流程的时间线]"
```

**Slide 3：现有方案的局限**
```
标题："Current Approaches Fall Short"

内容框架：
- 方法A：优点/致命缺陷
- 方法B：优点/致命缺陷
- 空白：我们的机会

示例：
"现有方法：
• 分子动力学模拟：准确但慢（数周）
• 机器学习：快但不准（泛化差）
• 我们需要：又快又准的方法"
```

**话术模板：**
```
"让我快速设定背景。
[问题] 在生物AI领域，我们面临一个核心挑战：____
[重要性] 这很重要，因为____（影响/成本/机会）
[现状] 目前的方法要么____，要么____
[缺口] 这留下了一个关键空白：____
这就是我们工作的起点。"
```

**注意事项：**
- 背景不要超过3分钟（20分钟talk的15%）
- 每个陈述都要有视觉支撑（图/表/示意图）
- 避免文献综述式的罗列
- 确保非专家能跟上（测试：本科生能理解吗？）

---

### 第四步：方法核心（5-6分钟）

**时间节点：** 第4-7张slides，talk的核心部分

**具体做法：**

**原则：单线程叙事，逐层展开**

**Slide 4：核心洞察（The Key Insight）**
```
标题："Our Key Insight"或"The Breakthrough"

内容：
- 一句话核心想法（大字，中间）
- 一个类比或视觉隐喻
- 为什么这个想法有效的直觉解释

示例：
"核心洞察：
蛋白质折叠不是搜索问题，是约束满足问题

[类比图：拼图 vs 数独]

传统方法：尝试所有可能的形状（搜索）
我们的方法：利用已知约束快速排除（推理）"
```

**Slide 5-6：方法架构**
```
标题："How It Works"

内容框架（选择一个可视化方式）：
1. 流程图（输入→处理→输出）
2. 三步法（Step 1/2/3，每步一个icon）
3. 层次图（从原始数据到最终结果的抽象层次）

关键：
- 最多3-4个模块
- 每个模块一句话解释
- 用颜色/箭头引导视线流动
- 突出"新"的部分（与现有方法的差异）

示例（三步法）：
Step 1: 数据编码
"我们将蛋白质序列转换为几何约束图"
[图示：序列→图]

Step 2: 约束推理
"使用图神经网络传播约束信息"
[图示：GNN示意图]

Step 3: 结构生成
"通过能量最小化生成3D结构"
[图示：2D→3D]
```

**Slide 7：技术亮点（可选）**
```
如果有特别创新的技术点：
- 用一张slide深入一个关键技术
- 但要保持可理解性（避免公式）
- 使用"这让我们能够____"的框架

示例：
"技术亮点：自适应采样策略

传统：均匀采样（浪费计算）
我们：集中计算在不确定区域
[对比图：热力图显示计算分布]

结果：速度提升10x，精度不降"
```

**话术模板：**
```
"现在让我解释我们的方法。

[核心洞察]
我们的关键洞察是：____
可以这样理解：[类比]

[架构]
整个方法分为三步：
第一，我们____（目的：____）
第二，我们____（这是创新点）
第三，我们____（得到最终结果）

[亮点]
这里有个技术细节值得一提：____
这让我们能够____（优势）

接下来看看这在实际中的表现。"
```

**注意事项：**
- 抵制"展示所有技术细节"的冲动
- 每张slide只讲一个点
- 使用"这意味着"来连接技术和影响
- 准备"如果有人问细节"的备用slides（放附录）
- 时刻检查：听众的眼睛还在跟随吗？

**能量管理：**
- 这是最容易失去听众的部分
- 每张slide切换时，改变语调或移动位置
- 使用手势指向屏幕上的关键元素
- 每2分钟插入一个"这很酷的地方是____"

---

### 第五步：结果展示（4-5分钟）

**时间节点：** 第8-11张slides

**具体做法：**

**原则：从具体到一般，从视觉到数字**

**Slide 8：代表性案例**
```
标题："It Works: A Real Example"

内容：
- 一个具体的、可视化的成功案例
- 前后对比（问题→我们的解决→结果）
- 包含"哇"时刻

示例：
"案例：预测COVID-19刺突蛋白结构

[左图] 实验结构（金标准）
[右图] 我们的预测
[中间] 重叠对比（RMSD = 0.8Å）

完成时间：2小时（传统方法需要数周）"
```

**Slide 9-10：系统性评估**
```
标题："Systematic Evaluation"

内容框架：
- 基准数据集
- 对比基线（2-3个主要竞品）
- 核心指标（1-2个，不要超过3个）
- 可视化（条形图/散点图/表格）

关键：
- 使用颜色突出你的方法
- 包含误差棒/置信区间
- 标注统计显著性（如果有）

示例：
[条形图]
Y轴：准确率（%）
X轴：方法A | 方法B | 方法C | 我们的方法

数据点：
方法A: 65% ± 5%
方法B: 72% ± 4%
方法C: 68% ± 6%
我们: 84% ± 3% ⭐

注释："在100个测试案例上，p < 0.001"
```

**Slide 11：额外优势**
```
标题："Beyond Accuracy"或"Additional Benefits"

内容：
- 速度/成本/可扩展性等维度
- 意外发现/副产品
- 用户反馈（如果有）

示例：
"额外优势：
• 速度：100x faster than MD simulations
• 成本：在单个GPU上运行（vs 超算集群）
• 可解释性：输出置信度分数和关键约束

[引用框]
'这改变了我们的工作流程' 
- 某制药公司研究员"
```

**话术模板：**
```
"结果如何？让数据说话。

[案例]
先看一个具体例子：____
[描述案例背景]
我们的方法____（结果）
这个结果____（为什么impressive）

[系统评估]
在更大规模的测试中：
我们在____数据集上评估
对比____（基线）
结果显示____（核心发现）
[指向图表] 这里可以看到____

[额外优势]
除了准确性，我们还发现：
____（速度/成本/其他）
这意味着____（实际影响）

这些结果让我们相信____（过渡到意义）"
```

**注意事项：**
- 诚实展示：包括失败案例或局限（增加可信度）
- 避免"cherry-picking"的印象
- 如果结果不是最好的，强调其他优势（速度/成本/可解释性）
- 准备好回答"为什么比X好"的问题

**视觉设计：**
- 图表要大（占slide的60-70%）
- 使用一致的配色方案
- 标注要清晰（字体至少24pt）
- 避免3D图表（难以解读）

### 第六步：意义阐述（2-3分钟）

**时间节点：** 第12-13张slides

**具体做法：**

**原则：从技术到影响，从我们到社区**

**Slide 12：直接影响**
```

内容框架（三层影响）：

"更大的图景：

2020: AlphaFold 2

开放问题：

如何处理____？"
```

**话术模板：**
```
"这为什么重要？

[直接影响]

[愿景]

[描述趋势]

**注意事项：**
- 不要过度承诺（"这将治愈癌症"）
- 平衡乐观和现实
- 连接到听众的研究兴趣（如果你做了情报工作）
- 为Q&A埋下"钩子"（开放性问题）

**创业者特殊考虑：**
如果你想暗示商业潜力但不想太明显：

[停顿，让听众自己推断商业价值]

### 第九步：Q&A主导（5-10分钟）

**时间节点：** Talk结束后，主持人开放提问

**具体做法：**

**开场框架：**
```
[主持人："现在开放提问"]

你的回应：
"谢谢！我很期待你们的问题。
[停顿2秒，扫视全场]
谁有第一个问题？"

[如果沉默超过5秒]
"或者，我可以先回答一个我经常被问到的问题：
'这个方法在____情况下表现如何？'
[简短回答]
但我更想听听你们的问题。"
```

**回答框架（AREA法）：**

**A - Acknowledge（确认）**
```
"好问题！"
"这是个重要的点。"
"很高兴你提到这个。"
"这正是我们也在思考的。"
```

**R - Reframe（重构，如果需要）**
```
"如果我理解正确，你是在问____？"
"让我确保我理解了你的问题：____"
[对于复杂/模糊的问题]
```

**E - Explain（解释）**
```
"简短回答是____
让我详细说明：____"

[结构化回答]
"有两个方面：
第一，____
第二，____"
```

**A - Anchor（锚定到你的核心信息）**
```
"这回到了我们的核心洞察：____"
"这正是为什么____（你的方法）很重要。"
```

**常见问题类型与应对：**

**类型1：技术细节问题**
```
问："你们用的是什么优化算法？"

回答：
"好问题。我们尝试了____和____，
最终选择了____，因为____（原因）。
[如果有备用slide] 我这里有张slide显示对比。
[如果没有] 具体细节在论文的____部分，
会后我可以详细解释。"
```

**类型2：对比竞品问题**
```
问："这和[某方法]有什么不同？"

回答：
"很好的对比。[某方法]的优势是____，
我们的方法在____方面不同，
具体来说____（关键差异）。
它们实际上是互补的，适用于不同场景。"

[避免直接贬低竞品]
```

**类型3：局限性/失败案例问题**
```
问："这在____情况下会失败吗？"

回答：
"诚实地说，是的。我们发现在____情况下，
性能会下降，因为____（原因）。
这是我们未来工作的一个重点方向。
[如果有解决思路] 我们正在探索____来解决这个问题。"

[诚实增加可信度]
```

**类型4：应用/扩展问题**
```
问："这能用于____吗？"

回答：
"有趣的想法！我们还没有在____上测试，
但原则上____（可行性分析）。
主要挑战可能是____。
如果你在这个领域，我很想会后聊聊，
看看是否有合作机会。"

[转化为合作机会]
```

**类型5：哲学/大问题**
```
问："这对AI的未来意味着什么？"

回答：
"这是个大问题。我的看法是____（简短观点）。
但具体到我们的工作，我认为____（连接回你的工作）。
这个话题很深，我很乐意会后深入讨论。"

[不要陷入长篇大论]
```

**类型6：挑战性/怀疑性问题**
```
问："我不确定这是否真的比____好。"

回答：
"我理解你的怀疑，这是健康的科学态度。
让我说明为什么我们相信____（证据）。
[展示数据]
当然，我们的评估可能有局限，
如果你有具体的对比场景，我很想听听。"

[保持专业，不要防御]
```

**类型7：无法回答的问题**
```
问："[超出你知识范围的问题]"

回答：
"这是个好问题，但我得诚实说，
这超出了我的专业领域。
我的理解是____（如果有部分了解），
但我不想误导你。
[如果知道专家] ____可能是更好的咨询对象。
或者我可以帮你找到相关资源。"

[诚实>瞎编]
```

**主导技巧：**

**技巧1：控制时长**
```
如果回答超过1.5分钟：
"这是个复杂的问题，我可以讲很久，
但为了给其他人机会，让我简短总结：____
会后我们可以深入讨论。"
```

**技巧2：重定向**
```
如果问题偏离主题：
"这是个有趣的方向，虽然不是今天的重点，
但____（简短回应）。
让我们先聚焦在____，然后会后可以聊这个。"
```

**技巧3：利用提问者**
```
如果提问者是该领域专家：
"这个问题很好，我注意到你在____方面有很深的研究。
[简短回答]
我很想听听你的看法，你觉得____？"

[转化为对话，展示谦逊]
```

**技巧4：埋下合作种子**
```
对于有潜力的提问者：
"这个问题表明你在____方面有深入思考。
[回答问题]
我们正在寻找____（合作/数据/应用场景），
如果你感兴趣，会后聊？"
```

**肢体语言：**
- 提问时，转向提问者，眼神接触
- 回答时，先看提问者，然后扫视全场（让所有人都参与）
- 使用手势强调要点
- 保持开放姿态（不要交叉双臂）

**时间管理：**
```
如果主持人说"最后一个问题"：
- 选择一个能强化核心信息的问题
- 回答要简洁有力
- 结尾："感谢所有精彩的问题，我会在外面，欢迎继续交流！"

如果没人提问：
- 不要惊慌
- 使用预设问题："一个常见问题是____"
- 或者："如果没有问题，我想强调____（核心信息）"
```

**注意事项：**
- 不要说"这是个愚蠢的问题"（即使是开玩笑）
- 不要打断提问者
- 不要与提问者争论（即使他们错了）
- 如果多个人同时举手，说"我先回答这个，然后是你"（指向下一个）

### 第十步：会后互动（Talk后30分钟）

**时间节点：** Q&A结束后，立即执行

**具体做法：**

**立即行动（前5分钟）：**

1. **不要立即离开舞台**
```
- 留在台上或台前
- 整理slides的同时，保持可接近状态
- 如果有人走近，优先他们而不是收拾东西
```

2. **主动接近关键人物**
```
如果你在情报收集阶段识别了关键人物：
- 走向他们："嗨，我注意到你在____方面的工作，
  很高兴你来听我的talk。你觉得____？"
- 不要等他们来找你（他们可能害羞或忙碌）
```

3. **处理排队者**
```
如果有多人等待交流：
- 对每个人："感谢你的关注，我想和每个人都聊，
  让我先快速回答这个问题，然后是你。"
- 每个对话控制在2-3分钟
- 对于深入讨论："这个话题很好，但我不想占用其他人的时间，
  我们能交换联系方式，之后详细聊吗？"
```

**对话框架（快速筛选）：**

**开场（10秒）：**
```
他们："很棒的talk！"
你："谢谢！你是做____的吗？"
[快速了解背景]
```

**评估（20秒）：**
```
判断对话价值：
- 潜在合作者？（相关研究/资源）
- 潜在用户？（应用场景）
- 潜在招聘对象？（学生/博后）
- 一般兴趣？（礼貌交流）
```

**分流（30秒）：**

**高价值对话：**
```
"这太有趣了，我们应该深入聊。
现在有点吵，你有空____（晚餐/coffee）吗？
或者我们交换邮件，我发给你____（材料/论文）？"

[立即交换名片或扫码]
[在名片背面写关键词，提醒自己]
```

**中等价值对话：**
```
"很高兴你对这个感兴趣。
我这里有____（一页纸摘要/QR码），
包含更多细节和联系方式。
如果你有具体问题，随时邮件我。"
```

**低价值对话：**
```
"谢谢你的反馈！
如果你想了解更多，论文在____（链接），
代码即将开源。"

[礼貌但简短]
```

**特殊情况处理：**

**情况1：有人想要合作**
```
他们："我们有____（数据/问题），想试试你的方法。"

你的回应：
"太好了！让我快速了解：
- 你的具体需求是____？
- 时间线是____？
- 你期望的输出是____？

[评估可行性]

如果可行：
'这听起来很匹配。让我们____（下一步）。
我会在____（时间）内联系你，我们详细讨论。'

如果不确定：
'这很有趣，但我需要了解更多细节。
你能发邮件给我，包括____（具体信息）吗？
我会评估后回复你。'

如果不可行：
'这个方向很有意思，但可能不太适合我们当前的方法，
因为____。不过____（替代建议/推荐其他人）。'"
```

**情况2：有人质疑你的结果**
```
他们："我不确定你的对比是公平的。"

你的回应（保持专业）：
"我理解你的关注。让我说明我们的对比设置：____
[解释方法]

你具体担心的是哪个方面？
[倾听]

[如果他们有道理]
'这是个好点，我们确实应该____（改进）。
感谢你指出这个，我会在后续工作中考虑。'

[如果他们误解了]
'我可能没有表达清楚。实际上____（澄清）。
这在论文的____部分有详细说明，我可以发给你。'

[如果他们只是想争论]
'我理解我们可能有不同看法。
科学就是这样进步的。
如果你有具体的实验建议，我很乐意听。'"
```

**情况3：学生想加入你的团队**
```
他们："我对这个方向很感兴趣，你们招人吗？"

你的回应：
"很高兴你感兴趣！告诉我一些你的背景：
- 你现在在____（学校/公司）？
- 你的研究兴趣是____？
- 你在找____（博后/实习/全职）？

[评估匹配度]

如果可能匹配：
'我们确实在____（方向）有机会。
发你的CV和简短的研究陈述到____（邮箱），
我会仔细看并回复你。'

如果不匹配：
'我们目前的重点是____，可能不太匹配。
但____（推荐其他实验室/公司）可能更合适。'

如果不确定：
'发你的材料给我，我看看是否有合适的机会。
即使现在没有，我也可以留意未来的可能性。'"
```
